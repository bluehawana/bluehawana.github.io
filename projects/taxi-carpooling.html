Gothenburg TaxiCarPooling Web Application

An intelligent carpooling platform using Spring Boot backend and Node.js microservices

TaxiCarPooling Platform

Project Overview

Gothenburg TaxiCarPooling is an innovative platform designed to optimize taxi usage in urban areas by matching passengers traveling in similar directions. The system reduces costs for passengers, increases revenue for drivers, and contributes to environmental sustainability by reducing the number of vehicles on the road.

Technical Details

  • Backend: Spring Boot, Node.js microservices
  • Frontend: React Native mobile application
  • Database: PostgreSQL
  • APIs: RESTful APIs, real-time data processing
  • Integrations: Payment gateways, geolocation services
  • Testing: Comprehensive automated testing suite

Project Description

This project is currently under development as a personal initiative to address urban transportation challenges in Gothenburg. The platform aims to create a more efficient and sustainable transportation ecosystem by leveraging modern technology to optimize taxi usage.

Key Features

  • Intelligent matching algorithm that pairs passengers based on routes and schedules
  • Real-time tracking and notifications for both passengers and drivers
  • Secure payment processing with fare splitting capabilities
  • Rating and review system to maintain service quality
  • Analytics dashboard for usage patterns and optimization opportunities

Technical Architecture

The system is built on a microservices architecture with the following components:

  • User Service: Handles user registration, authentication, and profile management
  • Matching Service: Implements the core algorithm for pairing passengers
  • Routing Service: Optimizes routes based on current traffic conditions
  • Payment Service: Manages secure transactions and fare calculations
  • Notification Service: Delivers real-time updates to users
  • Analytics Service: Processes usage data for insights and optimization

Current Status

The project is currently in active development, with the core matching algorithm and user management systems completed. The mobile application frontend is approximately 70% complete, with ongoing work on the payment integration and analytics dashboard.

Future Plans

Future development plans include:

  • Integration with public transportation APIs for multimodal journey planning
  • Implementation of machine learning for predictive demand modeling
  • Expansion to additional cities in Sweden
  • Development of a driver-specific mobile application
