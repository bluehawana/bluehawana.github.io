iPad Remote Development - No SSH or Terminal Apps Required

Hej,
Hope you all had a great Christmas. As a New Year's gift to the dev community, I want to share something I've been working on.

Today I successfully connected to my VPS from my iPad Pro Gen 3 and started coding - directly from Safari. No SSH apps, no terminal emulators, just a browser.

Tested over Tele2 cellular 4G connection - proving this works anywhere you have mobile data.

Solving iPad's Development Problem

The main reasons developers avoid iPad for serious work:
- No proper terminal tools
- Limited code IDE support
- Poor SSH connection options
- iOS sandboxing restrictions

This project solves all of these by moving the development environment to a server and accessing it through Safari. Now your iPad becomes a true development machine.

¬†What Works

¬†- Full terminal access in Safari
¬†- Git workflow (commit, push, PR creation)
¬†- File editing with vim/nano
¬†- AI-assisted coding (OpenAI integration)
¬†- Virtual keyboard optimized for touch
¬†- Sessions persist across disconnects
¬†- MFA with Google Authenticator for highest security

Built with Kiro and Claude Code Sonnet 4.5

This entire project was built using Kiro and Claude Code Sonnet 4.5 - proving that AI-assisted development can create real, production-ready tools. Now it enables vibe coding anywhere you have your iPad and internet connection. Coffee shop, airport, wherever - just open Safari and code.

Why This Matters

¬†The iPad Pro has the hardware for serious development work, but iOS limitations have always been the blocker. By running the dev environment on a server and accessing through Safari, we bypass those limitations completely.

You get:
- Full Linux environment
- All your command-line tools
- Your entire workflow, just accessible from anywhere
- No need to carry a laptop
- Secure access with multi-factor authentication

Technical Stack

- WebSocket for real-time communication
- xterm.js for browser terminal
- node-pty for shell management
- TypeScript monorepo
- PWA support
- TOTP-based MFA (Google Authenticator)

Next Steps

I'm developing several deployment scenarios for devs with(out) a VPS:

1. With VPS (what I'm using now)
2. Tailscale VPN (connect iPad to your Mac at home, no VPS needed)
3. Cloudflare Tunnel (permanent setup with custom domain)
4. ngrok (quick testing)
5. GitHub Codespaces (backup option)

For developers without a VPS, options 2-5 mean you can still work remotely with just your Mac at home and your iPad.

Open Source

Will be releasing this for the community soon. Whether you have a VPS, a Mac at home, or just use Codespaces - you'll be able to vibe code from your iPad anywhere you have internet. Forget your MBP and MBA, now in 2026, it is time to work from anywhere with just an iPad.
